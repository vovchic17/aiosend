# Security Policy

Security is very important to the **aiosend** project and its community. 🔒

Learn more below. 👇

## Supported Versions

The **latest version** of aiosend is supported.

It is strongly recommended to write tests for your application and regularly
update aiosend **after ensuring that all tests pass successfully**.  
This way, you can benefit from new features, bug fixes, and, most
importantly, **security updates**.

You can learn more about aiosend versions and how to pin and update them in
your project in the documentation.

## Vulnerability Disclosure

If you discover a vulnerability — **even if you are not sure that it is a real
security issue**, please report it as soon as possible.

To report a vulnerability, please contact us via the Telegram group:
**[@aiosend](https://t.me/aiosend)**

Moderators and community members will help to:

- verify the issue;
- reproduce it;
- properly document the report;
- find a safe and appropriate solution.

Please try to provide as much detail as possible, including reproduction steps
and example code, if available.

## Public Discussions

Please **avoid publicly disclosing security vulnerabilities** until the issue
has been reviewed and resolved. 🙊

It is better to discuss the problem within the community first to minimize
potential impact.
